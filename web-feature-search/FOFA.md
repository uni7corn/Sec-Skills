## 基础检索语法
FOFA核心检索基于**字段=值**的基础格式，所有值均需使用英文双引号包裹，支持IPv4/IPv6、域名、端口等基础资产维度检索，是所有复杂查询的基础。
```fofa
# IP地址精准检索
ip="192.168.1.1"
# IP网段检索（C段/掩码）
ip="10.0.0.0/24"
# 端口精准检索
port="8080"
# 根域名检索（匹配所有子域名）
domain="example.com"
# 网页标题检索
title="后台管理系统"
# URL/主机名片段检索
host=".gov.cn"
```
**核心要点**：基础字段为FOFA最常用检索维度，单字段检索可快速定位目标资产，无权限限制，免费版/付费版均可使用。

## 逻辑运算语法
通过逻辑运算符组合多个基础检索条件，实现多维度精准筛选，运算符区分大小写，需使用英文符号，优先级可通过英文括号`()`自定义。
```fofa
# 逻辑与（同时满足多个条件）
port="80" && title="登录"
# 逻辑或（满足任一条件）
port="80" || port="443"
# 逻辑非（排除指定条件）
os="Linux" && !port="22"
# 优先级自定义（先算括号内条件）
(port="80" || port="443") && country="CN" && title*="管理"
```
| 运算符                                                       | 含义       | 优先级 | 替代写法 |
| ------------------------------------------------------------ | ---------- | ------ | -------- |
| `&&`                                                         | 逻辑与     | 中     | `AND`    |
| `||`                                                         | 逻辑或     | 低     | `OR`     |
| `!`                                                          | 逻辑非     | 高     | `NOT`    |
| `()`                                                         | 优先级括号 | 最高   | -        |
| **核心要点**：无括号时，逻辑非`!`优先计算，其次是逻辑与`&&`，最后是逻辑或`||`；复杂查询建议用括号明确优先级，避免检索结果偏差。 |            |        |          |

## 匹配模式语法
针对字段值提供**精准/模糊/正则/不匹配**四种检索模式，适配不同检索需求，部分模式需付费版权限支持，是实现灵活检索的关键。
```fofa
# 精准匹配（完全一致，默认模式，可省略==）
title=="后台管理系统"
# 模糊匹配（包含指定字符串，付费版）
title*="管理"
# 正则匹配（按正则规则匹配，高级付费版）
title~="^后台.*登录$"
# 不匹配（排除包含指定值的资产）
os!="Windows Server"
```
| 模式                                                         | 语法 | 权限要求       | 适用场景                         |
| ------------------------------------------------------------ | ---- | -------------- | -------------------------------- |
| 精准匹配                                                     | `==` | 所有版本       | 已知准确值的精准检索             |
| 模糊匹配                                                     | `*=` | 个人版及以上   | 仅知关键词的模糊检索             |
| 正则匹配                                                     | `~=` | 高级会员及以上 | 复杂规则的精准匹配（无body支持） |
| 不匹配                                                       | `!=` | 所有版本       | 排除指定特征的资产               |
| **核心要点**：`=`与`==`功能一致，均为精准匹配；正则匹配不支持`body`字段，模糊匹配是日常使用最频繁的进阶模式。 |      |                |                                  |

## 高级检索字段
基于基础字段扩展的进阶检索维度，适配**服务指纹、应用系统、响应头**等精细化检索场景，可精准定位特定服务/应用的资产。
```fofa
# 多端口同时开放检索
ports="80,443,3389"
# Web服务器类型检索
server="Nginx/1.24.0"
# 应用系统检索（如OA、CMS）
app="致远OA"
# HTTP响应头检索
header="Server: Apache"
# 网页正文内容检索
body="管理员账号：admin"
# 服务横幅/指纹检索
banner="OpenSSH_8.4p1"
```
**核心要点**：`app`字段为FOFA自研的应用指纹识别，覆盖主流OA、CMS、中间件等；`banner`字段适配各类网络服务的指纹匹配，是端口服务检索的核心。

## 地理定位检索
基于资产的IP属地实现**国家/城市/运营商**维度的检索，适配地域化资产测绘需求，国家代码采用ISO 3166-1 alpha-2标准。
```fofa
# 中国境内资产检索
country="CN"
# 中国上海的资产检索
country="CN" && city="Shanghai"
# 中国移动运营商资产检索
isp="China Mobile"
```
| 字段                                                         | 含义      | 取值示例                           |
| ------------------------------------------------------------ | --------- | ---------------------------------- |
| `country`                                                    | 国家/地区 | CN（中国）、US（美国）、JP（日本） |
| `city`                                                       | 城市      | Beijing、Shanghai、Guangzhou       |
| `isp`                                                        | 运营商    | China Telecom、China Unicom        |
| **核心要点**：地理定位基于IP库解析，部分IP属地可能存在偏差；可结合基础字段实现“地域+特征”的精准测绘。 |           |                                    |

## 服务与协议检索
针对网络层/应用层协议实现精准检索，可定位特定协议的开放资产，适配**数据库、远程连接、物联网**等非Web服务的测绘。
```fofa
# 传输层协议检索（TCP/UDP）
base_protocol="tcp"
# 应用层协议检索
protocol="mysql" && port="3306"
# SSH协议检索
protocol="ssh" && banner="OpenSSH"
# FTP协议检索
protocol="ftp" && port="21"
```
| 常用协议                                                     | 端口示例 | 检索语法                             |
| ------------------------------------------------------------ | -------- | ------------------------------------ |
| HTTP/HTTPS                                                   | 80/443   | `protocol="http"`/`protocol="https"` |
| MySQL                                                        | 3306     | `protocol="mysql"`                   |
| Redis                                                        | 6379     | `protocol="redis"`                   |
| RDP                                                          | 3389     | `protocol="rdp"`                     |
| **核心要点**：`base_protocol`仅区分TCP/UDP，`protocol`为应用层具体协议，结合`port`字段可大幅提升检索精准度。 |          |                                      |

## 证书相关检索
针对HTTPS等加密服务的SSL/TLS证书实现检索，可通过**证书主体、颁发者、有效性**等维度定位相关资产，适配加密服务的测绘与合规排查。
```fofa
# 证书主体包含指定字符串
cert="example.com"
# 证书颁发者检索
cert.issuer="Let's Encrypt"
# 有效证书检索
cert.is_valid=true
# 证书过期检索
cert.is_valid=false
```
**核心要点**：`cert`字段匹配证书的主题、备用名称等核心字段；证书相关检索主要适配HTTPS资产，是排查过期证书、不合规加密服务的关键。

## 实战场景检索示例
基于上述语法，针对**资产测绘、漏洞挖掘、合规排查**三大核心场景，提供可直接复用的检索语句，覆盖日常工作中高频的FOFA使用场景。
### 资产测绘
```fofa
# 某企业全量资产（根域名+旗下IP）
domain="example.com" || ip="192.168.0.0/16"
# 中国境内Web资产（80/443端口）
country="CN" && (port="80" || port="443") && protocol="http"
# 某城市的Nginx服务器
country="CN" && city="Shenzhen" && server="Nginx"
```
### 漏洞挖掘
```fofa
# Redis未授权访问风险资产
port="6379" && protocol="redis" && banner!="requirepass"
# 开放RDP弱口令风险资产
country="CN" && port="3389" && protocol="rdp"
# 后台管理系统（易存在未授权/弱口令）
title*="后台" || title*="管理" || title*="登录"
```
### 合规排查
```fofa
# 暴露的Telnet明文服务
port="23" && protocol="telnet"
# 证书过期的HTTPS资产
protocol="https" && cert.is_valid=false
# 中国境内暴露的MySQL公网资产
country="CN" && port="3306" && protocol="mysql"
```

## 检索优化技巧
针对FOFA检索**结果过多、精准度低、检索缓慢**等问题，提供实用优化技巧，兼顾检索效率与结果精准度，适配免费版有限的检索资源。
1. **缩小检索范围**：优先添加`country`/`city`/`ip`等地域/IP条件，减少无关资产，例如`country="CN"`可过滤海外资产，大幅降低结果量；
2. **优先精准匹配**：日常检索优先使用`=`/`==`精准匹配，避免滥用模糊匹配`*=`，仅在未知准确值时使用，提升检索精准度；
3. **合理组合字段**：采用“**特征字段+基础字段**”的组合方式，例如`app="致远OA" && country="CN"`，而非单一字段检索；
4. **排除干扰资产**：使用`!`逻辑非排除测试资产、无关服务，例如`title!="测试" && !port="8080"`；
5. **控制字段数量**：单条检索语句字段数建议不超过5个，过多字段会导致检索缓慢，甚至无结果；
6. **利用更新时间**：结合FOFA网页端的“更新时间”筛选，仅检索近期存活的资产，避免无效资产。

**核心要点**：免费版FOFA有检索结果量和导出限制，优化检索语法可在有限资源内获取最有价值的资产。

## 常见语法错误排查
针对FOFA检索中高频出现的**语法报错、无结果、结果异常**等问题，梳理错误原因及解决方法，快速定位并解决检索问题。
### 语法报错
1. **引号错误**：使用中文双引号`“”`或单引号`''`，解决：统一替换为英文双引号`""`；
2. **运算符错误**：使用中文运算符`&&`/`||`（全角），解决：替换为英文半角运算符`&&`/`||`；
3. **字段拼写错误**：例如`titile`（正确`title`）、`domian`（正确`domain`），解决：核对FOFA官方字段列表，修正拼写；
4. **无意义空格**：字段与等号、等号与值之间加空格，例如`ip = "192.168.1.1"`，解决：删除空格，改为`ip="192.168.1.1"`。

### 检索无结果
1. **条件过于严格**：多个字段组合过于苛刻，例如`country="CN" && city="Beijing" && title="后台管理" && server="Nginx/1.24.0"`，解决：减少字段数量，放宽部分条件；
2. **使用无权限模式**：免费版使用模糊匹配`*=`或正则匹配`~=`，解决：升级付费版或改为精准匹配；
3. **字段值错误**：例如输入错误的IP、域名、端口，解决：核对目标资产的特征值，修正字段值。

### 结果异常（与预期不符）
1. **优先级未明确**：未使用括号导致逻辑运算顺序错误，例如`port="80" || port="443" && title="管理"`，解决：添加括号明确优先级，改为`(port="80" || port="443") && title="管理"`；
2. **混淆字段含义**：例如用`domain`检索子域名片段（正确用`host`），解决：区分`domain`（根域名）与`host`（URL片段）的差异；
3. **指纹识别偏差**：`app`/`server`字段匹配偏差，解决：结合`banner`/`header`字段辅助验证，提升精准度。

**核心要点**：FOFA检索语法对格式要求严格，出现问题时优先检查**引号、运算符、字段拼写**三大基础项。
