## 基础核心规则
Hunter检索的核心语法体系包含**匹配符、逻辑符、通配符**三类基础符号，搭配`字段="值"`的标准格式，是所有检索的基础，无权限限制，全版本通用。
```hunter
# 基础格式示例（包含匹配）
web.title="登录"
# 精确匹配示例
header.server=="Nginx/1.24.0"
# 逻辑组合示例
app.name="Tomcat" && port="8080" && province="北京"
# 通配符使用示例
domain="*.gov.cn"
```
### 核心符号说明
| 符号类型 | 符号/表达式       | 功能说明             | 适用场景                         |
| -------- | ----------------- | -------------------- | -------------------------------- |
| 匹配符   | `=`               | 包含匹配（模糊）     | 仅知关键词，需模糊检索           |
|          | `==`              | 精确匹配（完全一致） | 已知准确值，需精准定位           |
|          | `!=`              | 排除匹配（不包含）   | 过滤无关资产/特征                |
|          | `>`/`<`/`>=`/`<=` | 数值范围匹配         | 端口、版本号、状态码等数字型字段 |
| 逻辑符   | `&&`              | 逻辑与（且）         | 多个条件需同时满足               |
|          | `||`              | 逻辑或（或）         | 多个条件满足其一即可             |
|          | `!`               | 逻辑非（取反）       | 对单个条件进行排除               |
| 通配符   | `*`               | 匹配任意字符         | 域名、子域名等字符串片段匹配     |
### 格式规则
1. 标准格式：`字段="值"`，值包含特殊字符、空格、字符串时**必须用英文双引号**包裹
2. 简化格式：`字段==值`，值为纯数字、无特殊字符时可省略引号（如`port==8080`）
3. 优先级规则：`()` > `!` > `&&` > `||`，复杂组合建议用英文括号明确运算顺序

## 网站内容检索语法
针对网页可视化内容的专属检索维度，覆盖**标题、正文、相似资产、网站图标**等，是定位Web资产特征的核心语法。
```hunter
# 标题含登录关键词
web.title="登录"
# 正文含后台管理内容
web.body="后台管理"
# 匹配与目标域名特征相似的资产
web.similar="qianxin.com:443"
# 匹配相同网站图标的资产
web.icon="22eeab765346f14faf564a4709f98548"
```
| 语法指令                  | 功能说明                     | 示例                                          | 核心要点                                        |
| ------------------------- | ---------------------------- | --------------------------------------------- | ----------------------------------------------- |
| `web.title="关键词"`      | 网站标题包含指定关键词       | `web.title="后台管理系统"`                    | 检索速度快，是Web资产检索首选                   |
| `web.body="关键词"`       | 网页正文包含指定关键词       | `web.body="管理员账号：admin"`                | 正文内容量大，检索耗时略长，建议搭配其他条件    |
| `web.similar="域名:端口"` | 匹配与目标网站特征相似的资产 | `web.similar="baidu.com:443"`                 | 端口为必填项，仅支持80/443等常见Web端口         |
| `web.icon="MD5值"`        | 匹配相同网站图标的资产       | `web.icon="f9854822eeab765346f14faf564a4709"` | MD5值为32位小写，精准匹配，适用于同模板网站检索 |
| `web.tag="标签名"`        | 匹配带指定资产标签的资产     | `web.tag="登录页面"`                          | 标签为平台自定义，需先标注资产才能检索          |

## 域名检索语法
针对域名维度的检索，区分**域名片段、主域名后缀**，可精准筛选有域名标记的资产，适配域名相关资产测绘。
```hunter
# 域名包含指定片段
domain="qianxin.com"
# 精确匹配主域名（含所有子域名）
domain.suffix="qianxin.com"
# 仅筛选有域名的资产，排除纯IP资产
is_domain=true
```
| 语法指令                 | 功能说明                                   | 示例                        | 核心要点                                                     |
| ------------------------ | ------------------------------------------ | --------------------------- | ------------------------------------------------------------ |
| `domain="域名片段"`      | 域名包含指定字符串片段                     | `domain=".edu.cn"`          | 支持通配符`*`，如`domain="*.test.com"`匹配所有子域名         |
| `domain.suffix="主域名"` | 精确匹配主域名，包含其所有二级及以上子域名 | `domain.suffix="baidu.com"` | 仅匹配主域名为目标值的资产，如a.baidu.com、b.baidu.com均会命中 |
| `is_domain=true`         | 筛选带有域名标记的资产                     | `is_domain=true`            | 排除无域名的纯IP型资产，适配域名资产专项测绘                 |

## ICP备案检索语法
基于国内ICP备案信息的检索维度，覆盖**备案单位、网站名、备案号、备案类型**，是定位国内企业/机构合规资产的核心语法。
```hunter
# 备案单位名称含指定关键词
icp.name="奇安信"
# 备案网站名含指定关键词
icp.web_name="奇安信官网"
# 精确匹配ICP备案号
icp.number="京ICP备16020626号-8"
```
| 语法指令                | 功能说明                       | 示例                               | 核心要点                                           |
| ----------------------- | ------------------------------ | ---------------------------------- | -------------------------------------------------- |
| `icp.name="单位名"`     | 备案主体单位名称包含指定关键词 | `icp.name="腾讯科技"`              | 支持模糊匹配，需注意单位全称与简称的差异           |
| `icp.web_name="网站名"` | 备案的网站名称包含指定关键词   | `icp.web_name="微信官网"`          | 部分资产未备案网站名，可能导致检索结果为空         |
| `icp.number="备案号"`   | 精确匹配ICP备案号              | `icp.number="沪ICP备20250000号-1"` | 备案号需完整输入，平台对大小写不敏感               |
| `icp.type="类型"`       | 匹配备案主体的类型             | `icp.type="企业"`                  | 仅支持平台预设值：企业、个人、事业单位、政府机关等 |

## 应用组件检索语法
针对服务器端应用/组件的检索，覆盖**组件名、厂商、版本、分类**，可精准定位特定组件的资产，适配漏洞挖掘、组件合规排查。
```hunter
# 匹配指定组件名称
app.name="Tomcat"
# 匹配组件所属厂商
app.vendor="Apache"
# 匹配指定版本的组件
app.version="9.0.65"
```
| 语法指令               | 功能说明                             | 示例                   | 核心要点                                      |
| ---------------------- | ------------------------------------ | ---------------------- | --------------------------------------------- |
| `app.name="组件名"`    | 匹配指定的应用/中间件/数据库组件名称 | `app.name="MySQL"`     | Hunter自研指纹，覆盖主流开源/商业组件         |
| `app.vendor="厂商名"`  | 匹配组件的开发/所属厂商              | `app.vendor="Oracle"`  | 可批量检索某厂商的所有组件资产                |
| `app.version="版本号"` | 匹配指定版本的组件                   | `app.version="8.5.88"` | 支持数值范围匹配，如`app.version>="9.0"`      |
| `app.type="分类"`      | 匹配组件的行业分类                   | `app.type="中间件"`    | 支持分类：数据库、中间件、开发运维、CMS、OA等 |

## HTTP头服务检索语法
针对HTTP请求/响应头的检索，覆盖**通用头、Server头、响应状态码**，可定位特定Web服务、响应特征的资产。
```hunter
# HTTP头包含指定关键词
header="elastic"
# 精确匹配Server响应头
header.server=="Microsoft-IIS/10.0"
# 匹配HTTP响应状态码
header.status_code="200"
```
| 语法指令                      | 功能说明                        | 示例                                   | 核心要点                                                     |
| ----------------------------- | ------------------------------- | -------------------------------------- | ------------------------------------------------------------ |
| `header="关键词"`             | HTTP请求/响应头中包含指定关键词 | `header="Access-Control-Allow-Origin"` | 模糊匹配所有HTTP头字段，适配未知具体头字段的场景             |
| `header.server="服务名"`      | 匹配Server响应头的内容          | `header.server="Nginx"`                | 支持精确匹配`==`，精准定位特定Web服务器版本                  |
| `header.status_code="状态码"` | 匹配HTTP响应的状态码            | `header.status_code="404"`             | 仅支持数字型状态码，可搭配范围匹配如`header.status_code>="400"` |

## IP端口地理检索语法
针对网络层基础信息的检索，覆盖**单IP、IP段、端口、地理定位**，是资产测绘的基础语法，适配全网段/地域化资产筛选。
```hunter
# 精确匹配单个IP
ip="1.1.1.1"
# 匹配C段IP资产
ip="192.168.1.0/24"
# 匹配开放指定端口的资产
port="8080"
# 匹配中国上海的资产
country="中国" && city="上海"
```
| 语法指令          | 功能说明                       | 示例                | 核心要点                                           |
| ----------------- | ------------------------------ | ------------------- | -------------------------------------------------- |
| `ip="IP/IP段"`    | 精确匹配单个IP或指定网段的资产 | `ip="10.0.0.0/16"`  | 支持IPv4/IPv6，网段支持CIDR格式（/8/16/24等）      |
| `port="端口号"`   | 匹配开放指定端口的资产         | `port="3389"`       | 支持数值范围匹配，如`port>="8000" && port<="9000"` |
| `country="国家"`  | 匹配资产所在的国家/地区        | `country="中国"`    | 地理定位基于IP库解析，部分IP属地可能存在微小偏差   |
| `province="省份"` | 匹配资产所在的省级行政区       | `province="广东省"` | 仅适用于国内资产，省份名称需与平台预设一致         |
| `city="城市"`     | 匹配资产所在的城市             | `city="深圳市"`     | 仅适用于国内资产，支持地级市/直辖市                |

## 其他常用检索语法
覆盖**网络协议、操作系统、SSL证书**等高频通用维度，补充基础检索场景，适配非Web服务、加密服务的资产测绘。
```hunter
# 匹配指定网络协议的资产
protocol="HTTP"
# 匹配指定操作系统的资产
os="Windows Server 2019"
# 匹配SSL证书含指定关键词的资产
cert="Let's Encrypt"
```
| 语法指令          | 功能说明                              | 示例                  | 核心要点                                                  |
| ----------------- | ------------------------------------- | --------------------- | --------------------------------------------------------- |
| `protocol="协议"` | 匹配资产的网络应用层协议              | `protocol="SSH"`      | 覆盖常见协议：HTTP/HTTPS/SSH/FTP/MySQL/Redis等            |
| `os="系统名"`     | 匹配资产的操作系统类型/版本           | `os="Linux CentOS 7"` | Hunter基于服务指纹识别，无指纹则无法匹配                  |
| `cert="关键词"`   | 匹配SSL/TLS证书中包含指定关键词的资产 | `cert="baidu.com"`    | 匹配证书的主题、颁发者、备用名称等核心字段，适配HTTPS资产 |

## 实战场景组合示例
基于上述基础语法，针对**企业资产测绘、登录页排查、组件漏洞挖掘、地域资产筛选**等高频实战场景，提供可直接复制复用的检索语句，覆盖日常工作核心需求。
### 企业备案资产测绘
```hunter
# 某企业全量ICP备案域名资产
icp.name="某科技有限公司" && is_domain=true
# 某政府单位备案的政务资产
icp.name="某市人民政府" && domain="*.gov.cn"
```
### 登录页/后台资产排查
```hunter
# 国内企业资产中的登录页
icp.type="企业" && (web.title="登录" || web.title="后台")
# 含管理员登录的Web资产
web.body="管理员登录" && (port="80" || port="443")
```
### 组件/漏洞挖掘专项
```hunter
# 江苏地区开放8080端口的Tomcat资产
app.name="Tomcat" && port="8080" && province="江苏省"
# 低于9.0版本的Nginx资产（含漏洞风险）
header.server="Nginx" && app.version<"9.0" && country="中国"
# 开放Redis端口的无域名资产
protocol="Redis" && port="6379" && is_domain!=true
```
### 地域/端口专项筛选
```hunter
# 北京地区开放3389/RDP端口的资产
country="中国" && province="北京" && port="3389"
# 全网开放21/FTP端口的纯IP资产
protocol="FTP" && port="21" && is_domain!=true
```
### 排除类精准检索
```hunter
# 主域名为gov.cn的资产，排除测试域名
domain.suffix="gov.cn" && domain!="test.gov.cn"
# 某企业备案资产，排除上海地区
icp.name="某企业" && city!="上海市"
```

## 检索优化技巧
针对Hunter检索中**结果过多、精准度低、检索耗时、结果为空**等问题，提供实用优化技巧，兼顾检索效率与结果精准度，适配免费版有限的检索资源。
1. **先缩范围，再筛特征**：优先使用`country`/`province`/`ip`/`icp.type`等维度缩小检索范围，再添加`web.title`/`app.name`等特征字段，大幅减少结果量
2. **精准匹配优先，模糊匹配为辅**：已知准确值时优先使用`==`精确匹配，未知值时再用`=`模糊匹配，避免滥用模糊匹配导致结果冗余
3. **合理使用括号，明确优先级**：复杂逻辑组合必须用`()`明确运算顺序，如`(port="80" || port="443") && web.title="登录"`，避免因优先级错误导致结果偏差
4. **控制字段数量，避免过度组合**：单条检索语句的字段数建议不超过5个，过多字段会导致检索缓慢甚至无结果，可分多次检索逐步缩小范围
5. **数值字段用范围匹配**：针对端口、版本号、状态码等数值字段，使用`>`/`<`等范围匹配替代多个`||`，如`port>="8000" && port<="9000"`替代`port="8080" || port="8081"`
6. **排除无关资产，提升精准度**：使用`!`排除测试域名、非目标地域、无关端口的资产，如`!domain="test.com"`/`!city="北京市"`

## 常见语法问题排查
针对Hunter检索中高频出现的**语法报错、无结果、结果与预期不符**等问题，梳理错误原因及快速解决方法，无需专业知识即可定位并修复。
### 语法报错（检索语句无法执行）
1. **引号错误**：使用中文双引号`“”`或单引号`''` → 解决：统一替换为**英文双引号**`""`
2. **符号错误**：使用中文逻辑符`＆＆`/`｜｜` → 解决：替换为**英文半角符号**`&&`/`||`
3. **格式错误**：字段与值之间加空格，如`ip = "1.1.1.1"` → 解决：删除所有无意义空格，改为`ip="1.1.1.1"`
4. **字段拼写错误**：如`web.titile`/`icp.num` → 解决：核对官方字段列表，修正为正确字段`web.title`/`icp.number`

### 检索无结果（语句执行成功，无资产返回）
1. **条件过于严格**：多个精准字段过度组合，如`icp.name="某企业" && web.title="后台" && app.name="Tomcat" && city="北京"` → 解决：减少字段数量，放宽1-2个非核心条件
2. **值输入错误**：如备案号、IP段、版本号输入错误 → 解决：核对目标资产的准确信息，修正字段值
3. **字段与场景不匹配**：如对纯IP资产使用`domain.suffix` → 解决：删除不匹配字段，或添加`is_domain=true`筛选域名资产
4. **无指纹匹配**：如小众组件使用`app.name`检索 → 解决：改用`header`/`body`等模糊匹配方式

### 结果与预期不符（有结果但非目标资产）
1. **优先级未明确**：未用括号导致逻辑错误，如`port="80" || port="443" && web.title="登录"` → 解决：添加括号`(port="80" || port="443") && web.title="登录"`
2. **混淆匹配符**：需精确匹配却用了`=`模糊匹配，如`header.server="Nginx"`匹配到所有含Nginx的服务 → 解决：改用精确匹配`header.server=="Nginx/1.24.0"`
3. **地理定位偏差**：部分IP属地解析错误 → 解决：结合`isp`（运营商）字段辅助筛选，或放宽至省份维度
4. **指纹识别偏差**：`app.name`/`os`字段匹配到相似组件 → 解决：添加`header`/`banner`等字段辅助验证，提升精准度
